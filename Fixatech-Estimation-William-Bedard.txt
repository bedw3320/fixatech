DÉTAILS PAR PHASE

1. ADMIN & RENCONTRES (15-18h)

Approche: La découverte est en grande partie complétée. Heures restantes pour finaliser la validation du scope et révision de l'API ServiCentre.

Livrables:
• Validation finale des exigences
• Révision de la documentation API ServiCentre (Swagger)
• Plan d'intégration finalisé

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2. MIGRATION DE DONNÉES (50-60h)

Approche: Export CSV + application Matrixify pour l'importation dans Shopify.

Complexité:
• 4 000 produits (10x la moyenne des projets)
• Contenu bilingue (FR/EN)
• 5 paliers de prix B2B
• Métadonnées et spécifications techniques

Livrables:
• Export et transformation des données
• Mapping des colonnes Shopify
• Importation via Matrixify
• Validation et QA des données

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3. DÉVELOPPEMENT DE LA VITRINE (105-130h)

Approche: Personnalisation du thème Horizon à partir des designs Figma fournis.

Travaux inclus:
• Sections personnalisées (header, hero, carrousels, newsletter, footer)
• Composants UI (cartes produit, boutons, formulaires, recherche)
• Pages templates (PDP, collection, panier)
• Implémentation bilingue (FR/EN)
• Optimisation mobile et responsive
• Tests multi-appareils

Note: Un développeur Shopify standard prendrait 110-130h pour ce scope.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4. IMPLÉMENTATION B2B (40-45h)

Approche: Installation et configuration de l'application SparkLayer (bien intégrée avec Shopify).

Travaux inclus:
• Installation de l'application SparkLayer
• Configuration des groupes clients (5 paliers de prix)
• Personnalisation du portail B2B
• Gestion multi-utilisateurs et multi-adresses
• Configuration des termes de paiement (Net 30)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

5. INTÉGRATION ERP (55-85h)

Approche: Développement d'un middleware sur Google Cloud Platform avec connexion à l'API ServiCentre 360.

Pourquoi la fourchette est large:
L'API ServiCentre possède une documentation Swagger, mais la qualité et la complétude ne sont pas encore connues.

Meilleur scénario (55h): API propre et bien documentée
Pire scénario (85h): API complexe nécessitant des solutions de contournement

Travaux inclus:
• Développement du middleware (GCP ou plateforme similaire)
• Connexion aux endpoints ServiCentre 360
• Sync en temps quasi-réel:
  - Inventaire (ServiCentre → Shopify)
  - Prix par palier (ServiCentre → Shopify)
  - Commandes (Shopify → ServiCentre)
  - Statuts d'expédition (ServiCentre → Shopify)
• Gestion des erreurs et monitoring
• Tests d'intégration
• Documentation technique

⚠️ IMPORTANT: Si une solution middleware ou iPaaS existante est trouvée pendant la découverte, les coûts diminueront considérablement.

6. TESTS ET ASSURANCE QUALITÉ (15-25h)

Approche: Focus sur les tests du middleware. Le reste sera fait en collaboration avec l'équipe et le client avant le lancement.

Travaux inclus:
• Tests d'intégration middleware (8-10h)
• Tests de base de la vitrine (4-6h)
• Validation des cas limites (3-4h)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

7. FORMATION ET LANCEMENT (10-17h)

Approche: Fourniture d'assets pour auto-formation. Le lancement prend généralement 1 jour ouvrable + support le lendemain si tout se passe bien.

Livrables:
• Matériel de formation (guides, FAQ)
• Coordination du jour de lancement
• Support post-lancement (jour suivant)

HYPOTHÈSES ET RISQUES

Hypothèses clés:
✓ L'API ServiCentre 360 supporte la synchronisation en temps quasi-réel
✓ Les données produits sont propres et bien structurées
✓ Le contenu bilingue existe dans le système source
✓ Le plan Shopify Grow est suffisant (pas besoin de Plus)
✓ SparkLayer est compatible avec le plan Grow
✓ Le client fournit des retours en temps opportun

Facteurs de risque:
⚠️ API ServiCentre limitée → pourrait ajouter 10-20h de débogage
⚠️ Problèmes de qualité des données → pourrait ajouter 10-20h de nettoyage
⚠️ Complexité d'intégration plus élevée → la fourchette 50-80h absorbe ce risque
⚠️ Dérive du scope → SOW serré pour prévenir
